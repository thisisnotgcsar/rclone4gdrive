# Contributing to rclone4gdrive

All contributions, bug reports, and feature suggestions are welcome!  
All contributors will be added and referenced in the `README` file.

## Project Goals

Please, find under this repository's [issues](https://github.com/thisisnotgcsar/rclone4gdrive/issues) tab all the current needed contributions and TODOs.

> If any goal specification is unclear, please state it under the specific issue indicating what is not clear.

## Contribution Standards
- Produce well-documented code.
- Every file should have an initial summary comment explaining what the file does, why it is important, and (if applicable) which other files it works with.
- Commit messages should be concise and clearly indicate all micro-features implemented.
- Limit each commit to a few files and a few lines of codeâ€”avoid large, sweeping commits.

## How to Contribute
1. **Fork** this repository to your own GitHub account.
2. **Create a new branch** for your feature or bugfix.
3. Make your changes and **commit** them with clear but concise messages.
4. **Push** your branch to your forked repository.
5. **Open a Pull Request** from your branch to the main repository:
   - Go to your forked repository on GitHub.
   - Switch to your feature or bugfix branch.
   - Click the "New pull request" button.
   - Set the base repository to `thisisnotgcsar/rclone4gdrive` and the base branch to `main`.
   - Set the compare branch to your feature or bugfix branch.
   - Add a clear title and detailed description explaining your changes, referencing any related issues if applicable.
   - Review the changes shown, then click "Create pull request".
6. Wait for review and feedback from the maintainers.

Thank you for helping improve rclone4gdrive!
